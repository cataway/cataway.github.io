react中用得比较多的es6属性

发表于 2017-10-08 | 分类于 Work

import/export

import 在文件开头引入需要用到的模块或方法，在这个文件中导入的模块或方法是从另一个文件中导出的。
export default 导出的方法，可以直接定义其变量名，但是这样的方法每个文件只能导出一个：

export 导出的方法，可以是一个个的方法，在引入的时候，需要将方法包含在大括号里

const

es6的const用来声明常量，声明的时候必须就赋值，不然会报错，且一旦声明，变量的值就不能改变。
这里变量的值不能改变，实际上是指变量指向的内存地址不能改变。
基本数据类型（Number,String,Boolean），值保存在变量指向的内存地址，相当于一个常量，值不能更改。
引用数据类型（Object,Array），const声明的其实是一个只读的指针，指针的位置不能改变，但是其指向的值是可以操作的：

一般通过JSX创建的元素是不变的，通过const来声明一个React元素，而不用var

arrow function

使用箭头函数（arrow function）可以不用在意this的指向会变的问题
es5中，this指向问题：this关键字和严格模式，这篇文章有总结过
而箭头函数中的this，就是指向定义时所在的对象，不是函数执行时的对象
react中使用箭头函数，this会自动被绑定到函数中，就不用每次都通过bind手动绑定this了
