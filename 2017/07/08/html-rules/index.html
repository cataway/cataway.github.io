html规范

发表于 2017-07-08 | 分类于 Work

语法

  • 缩进使用tab（4个空格）
  • 嵌套的节点应该缩进
  • 在属性上，使用双引号，不要用单引号
  • 属性名全部小写，用中划线做分隔符
  • 不要在自动闭合标签结尾处加斜线（HTML5规范指出他们是可选的）
  • 不要忽略可选的关闭标签，eg：</li>、</body>

    HTML5 doctype

    在页面开头使用这个简单地doctype来启用标准模式，使其在每个浏览器中尽可能一致的表现
    虽然doctype不区分大小写，但是按照惯例，doctype大写

lang属性

根据HTML5规范：

应在html标签上加上lang属性。这会给语音工具和翻译工具帮助，告诉他们应当怎么去发音和翻译。
中文简体一般使用zh-cn

字符编码

通过声明一个明确的字符编码，让浏览器轻松、快速地确定适合网页内容的渲染方式，必须制定为UTF-8

IE兼容模式

用<meta>标签可以指定页面应该用什么版本的IE来渲染
不同doctype在不同浏览器下会触发不同的渲染模式，可参考这篇文章

引入CSS, JS

根据HTML5规范，通常引入CSS和JS不需要指明type，因为text/css和text/javascript分别是他们的默认值

属性顺序

属性顺序应按照特定的顺序以保证易读性

  • class
  • id
  • name
  • data-*
  • src, for, type, href, value, max-length, max, min, pattern
  • placeholder, title, alt
  • aria-*, role
  • requied, readonly, disabled
    class是为高可复用组件设计的，应处在第一位
    id更加具体且应该尽量少使用，应放在第二位

JS生成标签

在JS文件中生成标签让内容变得给你加难查找，难编辑，性能更差，应尽量避免这种情况的出现

减少标签数量

在编写HTML代码时，需要尽量避免多余的父节点
很多时候，需要通过迭代和重构来使HTML变得更少

实用高于完美

尽量遵循HTML标准和语义，但是不应该以浪费实用性为代价
任何时候都要用尽量小的复杂度和尽量少的标签解决问题
